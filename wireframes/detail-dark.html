# active milestone
Phase 2 is the cognitive core of Filogico. Getting MetaTrace right is not an instrumentation problem, it is a semantic design problem. The events it captures need to carry enough context to feed Fluid8 mastery scoring without becoming a firehose.

M5 — FluentTest Evaluation Framework Design

FluentTest is the validation and assessment intelligence layer. It evaluates correctness, process quality, and architectural reasoning simultaneously.

·Adaptive testing pipeline: dynamic complexity adjustment per learner mastery state from Fluid8
·AI-assisted evaluation: reasoning path analysis, not just final answer comparison
·Edge case generation: auto-generates test variants based on identified knowledge gaps

# decisions
Every architectural decision is logged here as a permanent record. The actual reasoning, the options considered, and why this path was chosen over the others.
ADR-007 open 2026-05-27

Event Mesh: NATS vs Kafka

NATS offers lower operational overhead for a small team. Kafka gives more replay and consumer group flexibility. The decision hinges on whether MetaTrace event replay is a first-class requirement or a nice-to-have.

ADR-006 decided 2026-05-24

Mastery Score: Dimensional vs Composite

Decided: dimensional. Eight discrete mastery scores instead of a single composite. Composite scores hide which dimension is failing and make adaptive trigger logic opaque.
